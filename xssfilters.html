Secure XSS Filters

Just Sufficient Output Escaping


Contextual Escaping Filters

XSS-Filters is the context-aware output filters for web applications. The design principle is to apply the filtering rules based on the output contexts in order to achieve the goal of “Just sufficient filtering”.

As shown in Figure 1, this means to escape the minimal set of characters to thwart JavaScript executions for the particular output contexts, thus preventing XSS attacks while keeping most characters intact. Compared to the traditional blindly escape filter, our filters are up to two times faster, and there is no more double-encoding problems like '&lt;'!!

Figure 1. "Just sufficient" encoding based on the HTML5 spec.

For manual usage, we provide a set of self-explanatory API(s) for the most common use cases. It is built to abstract away the details on how filters are chained, saving developers the pain to understand the numerous output contexts, and their execution order.

For automatic contextual esacping, please check out this tutorial for how to adopt the advanced automation solution.
